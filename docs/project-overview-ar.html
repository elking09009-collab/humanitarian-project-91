الشرح الشامل للمشروع الإنساني

وثيقة تعريفية تفصيلية بالمشروع، أهدافه، بنيته العامة، أقسامه التشغيلية، ودور كل جزء داخل المنظومة.

اسم المنصة المستخدم داخل الواجهة: حياة أفضل

1. فكرة المشروع

هذا المشروع عبارة عن منصة إنسانية رقمية متعددة الواجهات، هدفها تنظيم العمل الخيري والإغاثي بصورة عملية وقابلة للتوسع. المنصة تجمع بين استقبال الطلبات، عرض الاحتياجات، التبرع المالي والعيني، إدارة المتطوعين، التحقق الميداني، دعم المبادرات المجتمعية، ومتابعة الأثر داخل واجهة واحدة مترابطة.

المشروع لا يقتصر على عرض معلومات ثابتة، بل يضم جزءاً تنفيذياً حقيقياً عبر واجهات إدخال، مسارات API، قواعد بيانات، ولوحة إدارة تسمح بمتابعة البيانات وحوكمتها. كما يضم نسخة ويب عامة وواجهة خلفية إدارية، إلى جانب تطبيق موبايل مبدئي يخدم سيناريوهات تسجيل الدخول واستعراض الاحتياجات.

2. الأهداف الرئيسية

تنظيم وصول المساعدة

تجميع الاحتياجات في مكان واحد وربطها بمناطق جغرافية وحالات ميدانية قابلة للتتبع.

تسهيل التطوع

إتاحة التطوع العام، والتطوع التخصصي، والتحقق الميداني، وربط الأسر بالداعمين.

تنويع قنوات العطاء

دعم التبرعات المباشرة، الصناديق الجماعية، المشاريع الصغيرة، الأوقاف المصغرة، والوصايا الرقمية.

تعزيز الشفافية

الاعتماد على التوثيق، القصص، التعليقات، والتحقق الإداري قبل اعتماد بعض العمليات الحساسة.

رفع الكفاءة التشغيلية

تحويل البيانات إلى مسارات قابلة للقراءة والتحليل بدلاً من الاكتفاء بصفحات تعريفية منفصلة.

إتاحة الوصول من أكثر من قناة

واجهة ويب عامة، API خلفية، لوحة إدارة، وتطبيق موبايل مبدئي.

3. النظرة العامة المعمارية

المشروع مبني على بنية متعددة الطبقات، وكل طبقة لها دور واضح:

  • الواجهة الأمامية: صفحات HTML وCSS وJavaScript داخل مجلد frontend، وهي واجهة المستخدم العامة.
  • الواجهة الخلفية: مشروع Laravel داخل backend ويحتوي على الـ API، النماذج، المهاجرات، والاختبارات.
  • لوحة الإدارة: موجودة داخل Laravel عبر Filament لإدارة البيانات ومتابعة التشغيل.
  • تطبيق الموبايل: موجود داخل mobile ويوفر نسخة مبكرة من الوصول عبر الهاتف.
  • النشر: الواجهة الأمامية قابلة للنشر كصفحات ثابتة، بينما تعمل الواجهة الخلفية كسيرفر API منفصل.

4. أقسام المشروع الرئيسية ودور كل قسم

4.1 الواجهة العامة للمستخدم

هذا الجزء هو الواجهة المرئية للمشروع. يضم الصفحة الرئيسية وصفحات التعريف والخدمات والتبرع والتطوع وغيرها. دوره هو توجيه المستخدم، تقديم المعلومات، وجمع البيانات من خلال النماذج ثم إرسالها إلى الواجهة الخلفية.

الصفحات / الوحدات الدور
index.html المدخل الرئيسي للمنصة، يعرض الرسالة العامة وبعض المؤشرات والمداخل السريعة.
services.html عرض الخدمات المتاحة ومسارات التعامل مع المستفيدين والمتبرعين.
donation.html وصفحات الغذاء والدواء والملابس والدم والمال والدين تنظيم أشكال التبرع المختلفة وتوجيه المستخدم إلى نوع الدعم المناسب.
request.html وrequest-status.html تقديم الطلبات ومتابعة حالتها داخل المنظومة.
login.html وregister.html التسجيل والدخول وربط المستخدم بجلسة عمل وآلية التحقق.
about-us.html وcontact-us.html تقديم تعريف مؤسسي وقنوات التواصل المباشر.

4.2 قسم التطوع والمجتمع

هذا القسم يمثل منظومة العمل التطوعي والمجتمعي داخل المشروع. تم تصميمه بحيث لا يقتصر على استمارة عامة، بل يشمل أدواراً مختلفة ومخرجات متعددة.

الوحدة وظيفتها
volunteer.html بوابة التطوع العامة. تجمع بيانات المتطوعين، تخصصاتهم، ومدنهم، وتعرض إحصاءات مرتبطة بالنشاط العام.
volunteer-investigator.html مخصص للتوثيق الميداني ورفع تقارير التحقق على الاحتياجات القائمة، وهو مسار مختلف عن طلب التطوع العام.
skill-giving.html ربط المهارات المهنية والتخصصية بخدمة العمل الإنساني بدلاً من قصر العطاء على المال أو المواد.
human-twin.html تفعيل مفهوم التوأمة الإنسانية بين داعم وأسرة، مع الرسائل والإحصاءات والربط بحسب المدينة.
impact-wall.html منصة للقصص والمجتمع والتفاعل العام وإبراز أثر المساهمات الإنسانية.

4.3 قسم العطاء والتمويل

هذا الجزء يغطي وسائل التمويل الخيري المتنوعة داخل المشروع، ويهدف إلى توسيع مصادر الدعم وعدم الاكتفاء بنموذج تبرع واحد.

  • charity-funds.html: صناديق جماعية أو حملات ذات هدف واضح يمكن المساهمة فيها ومتابعة التقدم.
  • micro-projects.html: تمويل مشاريع متناهية الصغر لفئات مستحقة أو مبادرات إنتاجية صغيرة.
  • micro-endowments.html: نموذج استثماري خيري أو وقفي مصغر يهدف إلى الاستدامة.
  • gift-donation.html: تحويل العطاء إلى تجربة هدية أو مساهمة ذات طابع شخصي وإنساني.
  • legacy-giving.html: إدارة فكرة الوصية الرقمية أو التبرعات المؤجلة ذات الأثر المستقبلي.
  • donation.html: نقطة انطلاق عامة لاختيار نوع التبرع المناسب.

4.4 قسم الأصول والشراكات

هذا القسم يخدم إدارة الموارد غير النقدية والعلاقات المؤسسية، وهو مهم لتشغيل المنصة على مستوى أوسع من التبرع الفردي.

  • smart-inventory.html: إدارة الأصول العينية مثل المخزون والاحتياطي والتوزيع والحجز.
  • csr-portal.html: تسجيل وتفعيل مساهمة الشركات ضمن المسؤولية الاجتماعية.

4.5 قسم المتابعة الذكية والطوارئ

يجمع هذا القسم بين قراءة الوضع الإنساني، الاستجابة السريعة، ورؤية البيانات جغرافياً أو تحليلياً.

  • heatmap.html: عرض خريطة الاحتياج المجمعة حسب المناطق مع كثافة الحالات.
  • crisis-predictor.html: عرض تنبؤات أو إشارات مرتبطة بالأزمات المحتملة.
  • sos-emergency.html: واجهة للحالات العاجلة وسرعة الاستجابة.
  • chatbot.html: مدخل مساند للإجابة والتوجيه داخل النظام.

4.6 الواجهة الخلفية الإدارية

الواجهة الخلفية مسؤولة عن إدارة البيانات وقواعد العمل. داخلها توجد النماذج والمتحكمات والمهاجرات والمسارات، بالإضافة إلى لوحة Filament التي تسهّل إدارة الكيانات التشغيلية ومراقبة المدخلات.

  • إدارة المستخدمين والصلاحيات والحالة الإدارية للحسابات.
  • إدارة المناطق والاحتياجات والتعليقات والتحقق الميداني.
  • إدارة صناديق التبرع والمشاريع والوقف المصغر والأزمات والشراكات.
  • إدارة بيانات التوأمة الإنسانية والرسائل بين الداعم والأسرة.
  • توفير API موحد للواجهة العامة وتطبيق الموبايل.

4.7 تطبيق الموبايل

يوجد داخل المشروع تطبيق موبايل مبدئي مبني على Expo وReact Native. وظيفته الحالية تمثيل قناة وصول عبر الهاتف لسيناريوهات أساسية مثل تسجيل الدخول، التسجيل، استعراض الاحتياجات، وتفاصيل الحاجة.

5. هيكل البيانات والكيانات الأساسية

يعتمد المشروع على مجموعة كيانات رئيسية تشكل قلب المنظومة:

  • Users: المستخدمون بأدوار مختلفة مثل الإدارة والمتطوع والمستخدم العام.
  • Areas: المناطق الجغرافية المرتبطة بالاحتياج والخريطة.
  • Needs: الاحتياجات الأساسية مع نوع الحاجة والكمية والحالة والمنطقة.
  • Need Comments: تعليقات أو ملاحظات مرتبطة بكل احتياج.
  • Volunteer Validations: توثيق ميداني للحالات بعد زيارة فعلية.
  • Volunteer Applications: طلبات التطوع العامة الواردة من صفحة التطوع.
  • Twin Families / Twin Supporters / Twin Messages: كيانات نظام التوأمة الإنسانية.
  • Charity Funds / Micro Projects / Endowments / Emergency Cases / CSR: كيانات تشغيل الوحدات المتخصصة.

6. المسارات والخدمات البرمجية

طبقة الـ API تمثل العمود الفقري للربط بين الواجهة والبيانات. المشروع يحتوي على مجموعة واسعة من المسارات تغطي:

  • المصادقة: التسجيل، الدخول، الخروج، قراءة بيانات المستخدم الحالي.
  • المناطق والاحتياجات: عرض المناطق، عرض الاحتياجات، إضافة الاحتياجات، الحالات المعلقة، والتفاصيل.
  • التفاعل المجتمعي: التعليقات، القصص، المنتدى، الإعجابات.
  • التمويل: الصناديق، المشاريع، الأوقاف، القروض، التبرعات المرتبطة بالأزمات.
  • التطوع: التحقق الميداني، طلبات التطوع العامة، وخريطة الاحتياج.
  • التوأمة الإنسانية: تسجيل الأسر، تسجيل الداعمين، الإحصاءات، والرسائل.

8. آلية العمل العامة داخل المنصة

  1. يدخل المستخدم إلى الواجهة العامة ويختار الخدمة المناسبة: طلب، تبرع، تطوع، أو متابعة.
  2. ترسل الواجهة البيانات إلى الـ API بالصيغة المناسبة وفق نوع العملية.
  3. تقوم الواجهة الخلفية بالتحقق من المدخلات وتخزينها في قاعدة البيانات.
  4. تُعرض البيانات لاحقاً داخل الواجهات العامة أو الإدارية بحسب طبيعتها وصلاحيات الوصول.
  5. بعض المسارات عامة، وبعضها يتطلب تسجيل دخول وتوكن صالح.
  6. تتحرك بعض الوحدات لاحقاً من حالة الإدخال إلى حالة المراجعة ثم الاعتماد أو التنفيذ.

9. ملامح القوة في المشروع

  • تنوع كبير في الوحدات بدل الاقتصار على نموذج خيري واحد.
  • فصل واضح بين الواجهة العامة والواجهة الخلفية.
  • وجود مسارات API حقيقية واختبارات تشغيلية.
  • قابلية التوسع إلى لوحة إدارة وتطبيق موبايل دون إعادة بناء المشروع من الصفر.
  • تركيز على الشفافية من خلال التحقق الميداني والتفاعل المجتمعي والرسائل.

10. الحالة الحالية للمشروع

في حالته الحالية، المشروع يحتوي على واجهة أمامية موسعة، API خلفية متعددة المسارات، لوحة إدارة داخل Laravel، واختبارات أساسية ناجحة. كما تم فصل طلبات التطوع العامة عن مسار التحقق الميداني، وهو ما يحسن دقة منطق النظام ويمنع تضارب الوظائف بين نوعين مختلفين من العمليات التطوعية.

هذه الوثيقة تعريفية وتشغيلية، ومصممة لتصلح كملف تقديم للمشروع أو مرجع داخلي لفهم مكوناته ووظائفه.
